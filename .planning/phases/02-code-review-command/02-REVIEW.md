### IN-03: `globals.css` retains create-next-app theme variables that conflict with project direction

**File:** `src/app/globals.css:3-13,15-20,22-25`
**Issue:** The global CSS includes `create-next-app` default theming:
- Light/dark mode CSS variables (`--background: #ffffff`, `--foreground: #171717` and dark variants)
- A dark mode `prefers-color-scheme` media query (lines 15-20)
- Body fallback to `Arial, Helvetica, sans-serif` (line 25)
- `@theme inline` block maps `--font-geist-sans` and `--font-geist-mono` which come from the Geist font import in `layout.js`

The plan's Task 5 specifies a different design direction:
- Background: `#FAF8F5` (warm off-white)
- Text: `#2D2926` (dark brown-black)
- Font: Pretendard, system-ui, -apple-system, sans-serif

The current CSS will need to be mostly replaced. The Geist font variables and Arial fallback are unused by the plan's target design, and the dark mode media query is irrelevant for a branded publication app.

**Fix:** This is pre-empted by Task 5. No immediate action needed, but the dark mode block (lines 15-20) and Arial fallback (line 25) could be removed now to reduce noise:

```css
@import "tailwindcss";

@theme inline {
  --font-sans: 'Pretendard', system-ui, -apple-system, sans-serif;
  --font-mono: var(--font-geist-mono);
}

body {
  font-family: var(--font-sans);
}
```

## Additional Observations (Non-Issues)

### `next.config.mjs` — ESM vs plan's CommonJS specification
The plan specifies `next.config.js` with `module.exports = nextConfig`. The implementation uses `next.config.mjs` with `export default nextConfig`. This is the correct output for Next.js 16's `create-next-app`. Next.js supports both `.js` and `.mjs` config files. No issue.

### Next.js 15 vs 16 version discrepancy
The plan overview states "Next.js 15 (App Router)" but `create-next-app@latest` installed 16.2.9. The `AGENTS.md` warns about breaking changes. While the current scaffold builds cleanly (proven), downstream tasks' code examples from the plan are written for Next.js 15 patterns and may require adaptation. This is a **plan-level inconsistency**, not a Task 1 implementation bug, because Task 1 correctly executed the plan's own `create-next-app@latest` command.

### `package-lock.json` size (7332 lines)
Auto-generated lockfile. Size is proportional to the dependency tree (Next.js 16 + React 19 + Supabase clients + PDF renderer). Expected artifact. Not a code quality concern.
